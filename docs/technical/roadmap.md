# Roadmap

## Current Phase

The project is in a scope-correction phase.

The immediate task is to move the repository from its inherited legacy framing to documentation and modeling language that matches the actual AIMS-LEAF objective: AI/ML-driven integration of multiscale imaging and sequencing data for plant heat stress tolerance.

## Short-Term Priorities

### 1. Documentation Alignment

- remove outdated legacy framing from the published docs
- define the project around host response, multiscale integration, and predictive modeling
- document the intended refactor direction clearly

### 2. Schema Refactor Planning

- identify inherited classes that do not fit the new project scope
- define the core entities needed for genotype, treatment, assay output, phenotype, and prediction
- plan a migration path from inherited terminology to AIMS-LEAF terminology

### 3. AI/ML Workflow Readiness

- identify the minimal metadata required for training and evaluation datasets
- define provenance links between source data, derived features, and predictions
- support comparison of observed and predicted host outcomes

## Medium-Term Priorities

- formalize multiscale assay patterns for imaging and sequencing inputs
- add documentation for model artifacts and benchmarking conventions
- produce example datasets aligned to plant heat stress response use cases

## Success Criteria

The documentation refactor will be successful when the published site answers three questions clearly:

1. what biological problem AIMS-LEAF addresses
2. what kinds of data the project integrates
3. how the model supports predictive analysis of host response
